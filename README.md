# Synchronous Voter Model on Non‑Bipartite Graphs: Universal Consensus

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Python 3.9+](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/downloads/)

This repository contains all the code to reproduce the numerical experiments from the paper:

> **"Synchronous imitation on non‑bipartite graphs guarantees global consensus without an external observer"**  
> (preprint, 2026)

## Summary

We show that the **synchronous (parallel) voter model** on any connected **non‑bipartite** graph reaches **full consensus with probability 1** in time ~O(N^1.2).  
On bipartite graphs (e.g., a cycle), the dynamics either fixate (47% of runs) or fall into a persistent 2‑cycle (53% of runs).  

Key findings:
- Majority rule (synchronous) **fails** on regular graphs with degree k=2,3,4 (0% consensus), and gives only ~45% consensus for k=5.
- Asynchronous voter model always converges, but is slow (~N² steps).
- Synchronous voter on non‑bipartite graphs: 100% consensus in **hundreds of steps** (N=200) to thousands (N=500).
- Speed increases with degree and presence of hubs (fastest on Barabási–Albert: ~243 steps for N=200).

## Requirements

- Python 3.9+
- numpy
- scipy
- networkx
- pandas
- tqdm (for progress bars)

Install with:

```bash
pip install -r requirements.txt

Reproducing all results
Simply run the orchestration script:
python scripts/run_all_experiments.py

This will:

Generate all required graphs (random regular, Watts–Strogatz, Barabási–Albert, ring, irregular)

Run all 8 experiment blocks from the paper

Save aggregated results (CSV) into the results/ folder

Print summary tables to the console

Important: All random number generators are seeded (default seed = 42).
Running the script twice on the same machine should produce identical aggregated results.
Small deviations may occur due to floating‑point or library version differences, but the main claims (100% consensus on non‑bipartite graphs, ~47% on the ring) remain robust.

Running individual experiments
Each function inside src/ can be imported and used separately.
Example – run synchronous voter on a regular random graph with N=500, k=5, 30 runs:
from src.voter_sync import sync_voter_experiment
from src.graph_generators import regular_random_graph

results = sync_voter_experiment(
    graph_generator=regular_random_graph,
    N=500, k=5, runs=30,
    max_steps=100000, seed=42
)
print(results['consensus_rate'])   # should be 1.0

Data availability
Raw CSV files (e.g., per‑run trajectories) are not included in the repository to keep it lightweight.
They can be regenerated by running the experiments with save_detailed=True (modify the script accordingly).
The aggregated numbers reported in the paper's tables are reproduced by the script above.
